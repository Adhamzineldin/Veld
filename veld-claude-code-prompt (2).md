I am building **Veld** — a contract-first, multi-stack API code generator written in Go.

## Core Concept
Developer writes a `.veld` contract file once. Veld generates:
1. A typed frontend SDK (TypeScript) with full autocomplete
2. Backend service interfaces + route wiring for their chosen framework
3. Developer writes business logic in their own files — Veld NEVER touches those

**generated/routes/auth.routes.ts**
```typescript
// AUTO-GENERATED BY VELD — DO NOT EDIT
import { Router } from 'express'
import type { IAuthService } from '../interfaces/IAuthService'

export function authRouter(service: IAuthService) {
  const router = Router()

  router.post('/auth/login', async (req, res) => {
    const result = await service.Login(req.body)
    res.json(result)
  })

  router.post('/auth/register', async (req, res) => {
    const result = await service.Register(req.body)
    res.json(result)
  })

  router.get('/auth/me', async (req, res) => {
    const result = await service.Me((req as any).user.id)
    res.json(result)
  })

  router.post('/auth/logout', async (req, res) => {
    const result = await service.Logout((req as any).user.id)
    res.json(result)
  })

  return router
}
```

**generated/client/api.ts** (frontend SDK)
```typescript
// AUTO-GENERATED BY VELD — DO NOT EDIT
import type { LoginInput, RegisterInput, User, SuccessResponse } from '../types/auth'

const BASE = process.env.VELD_API_URL ?? ''

async function post<T>(path: string, body: unknown): Promise<T> {
  const res = await fetch(BASE + path, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body)
  })
  if (!res.ok) throw new Error(await res.text())
  return res.json()
}

async function get<T>(path: string): Promise<T> {
  const res = await fetch(BASE + path)
  if (!res.ok) throw new Error(await res.text())
  return res.json()
}

export const api = {
  Auth: {
    Login:    (input: LoginInput): Promise<User>             => post('/auth/login', input),
    Register: (input: RegisterInput): Promise<User>          => post('/auth/register', input),
    Logout:   (): Promise<SuccessResponse>                   => post('/auth/logout', {}),
    Me:       (): Promise<User>                              => get('/auth/me'),
  }
}
```

## Hard Rules
1. Parser and emitters are completely isolated — only AST JSON passes between them
2. Emitter interface in Go: `type Emitter interface { Emit(ast AST, outDir string) error }`
3. All generated files have `// AUTO-GENERATED BY VELD — DO NOT EDIT` header
4. Type mapping: veld `string` → TS `string`, veld `int` → TS `number`, veld `bool` → TS `boolean`
5. Output is deterministic — same input always produces identical output
6. Veld NEVER writes outside `/generated/` directory
7. Actions with no `input` field generate a handler with no body parameter

## Build Order
1. Define all AST structs in `internal/ast/ast.go`
2. Implement the Lexer — tokens for: `model`, `module`, `action`, `input`, `output`, `middleware`, HTTP methods (GET/POST/PUT/DELETE/PATCH), `{`, `}`, identifiers, paths like `/auth/login`, string types
3. Implement the recursive descent Parser
4. Implement the Validator — check all input/output type refs exist as defined models, detect duplicate action names, detect duplicate module names
5. Implement the Express + TypeScript emitters
6. Wire everything into the CLI with `cobra`

## veld init — Full Specification

# My Veld Project

## How it works

1. Edit `api.veld` to define your API contract
2. Run `veld generate` to regenerate the typed SDK and backend interfaces
3. Implement your business logic in `app/services/`
4. Import the generated SDK in your frontend from `generated/client/api.ts`

## Commands

| Command | Description |
|---------|-------------|
| `veld generate` | Generate from api.veld using veld.config.json |
| `veld generate --backend=flask --frontend=react` | Override stack |
| `veld validate` | Validate contract without generating |
| `veld watch` | Watch mode — regenerate on save |
| `veld ast` | Dump AST JSON for debugging |

## File ownership

| Directory | Owner |
|-----------|-------|
| `generated/` | Veld — do not edit manually |
| `app/` | You — never overwritten by Veld |
