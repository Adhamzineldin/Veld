# java-angular

Spring Boot + Angular example powered by Veld.

## What this shows

- Java/Spring Boot backend implementing the generated service interfaces with in-memory data
- Angular frontend using a service with the generated fetch SDK
- No database setup needed — data lives in `ArrayList` fields on each service

## Project layout

```
java-angular/
├── veld/               ← Veld contracts (source of truth)
├── generated/          ← Auto-generated by `veld generate` (do not edit)
├── backend/            ← Spring Boot app
│   ├── pom.xml
│   └── src/main/java/com/example/
│       ├── Application.java
│       └── services/
│           ├── UsersServiceImpl.java
│           └── TodosServiceImpl.java
└── frontend/
    └── todo.component.ts
```

## Setup

### 1. Generate the code

```bash
cd java-angular/veld
veld generate
```

This writes typed interfaces, route handlers, and the fetch SDK into `../generated/`.

### 2. Start the backend

Requires Java 17+ and Maven 3.8+.

```bash
cd java-angular/backend
mvn spring-boot:run
```

The server listens on `http://localhost:8080`.

### 3. Start the frontend

Create an Angular project in `frontend/`, install dependencies, then drop `todo.component.ts` in.

```bash
npm install -g @angular/cli
ng new frontend --standalone --routing=false --style=css
cd frontend
npm install
ng serve
```

Set `VELD_API_URL=http://localhost:8080` (or configure `baseUrl` in `veld.config.json`) so the generated SDK points at your running backend.

## How the wiring works

`veld generate` produces `generated/interfaces/IUsersService.java` and `generated/interfaces/ITodosService.java`. The `@Service` classes in `backend/src/main/java/com/example/services/` implement those interfaces directly. Spring Boot auto-discovers them and the generated route handlers call them via constructor injection.

On the Angular side, `todo.component.ts` injects a wrapper service that delegates to the generated `api` client from `generated/client/api.ts`.
