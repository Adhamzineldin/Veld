# node-react

Express + React (TanStack Query) example powered by Veld.

## What this shows

- Node/Express backend with generated route registration and typed service interfaces
- React frontend using TanStack Query hooks wrapping the generated `api` client
- In-memory data store so no database setup is needed

## Project layout

```
node-react/
├── veld/               ← Veld contracts (source of truth)
├── generated/          ← Auto-generated by `veld generate` (do not edit)
├── backend/            ← Express app
│   └── src/
│       ├── index.ts
│       └── services/
│           ├── UsersService.ts
│           └── TodosService.ts
└── frontend/
    └── App.tsx
```

## Setup

### 1. Generate the code

```bash
cd node-react/veld
veld generate
```

This writes typed interfaces, route handlers, Zod schemas, and the fetch SDK into `../generated/`.

### 2. Start the backend

```bash
cd node-react/backend
npm install
npm run dev
```

The server listens on `http://localhost:3000`.

### 3. Start the frontend

Create a Vite + React project in `frontend/`, install dependencies, then drop `App.tsx` in.

```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
npm install @tanstack/react-query
npm run dev
```

Set `VELD_API_URL=http://localhost:3000` (or configure `baseUrl` in `veld.config.json`) so the generated SDK points at your running backend.
