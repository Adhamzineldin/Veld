# csharp-kotlin

ASP.NET Core Minimal API + Kotlin Android client example powered by Veld.

## What this shows

- ASP.NET Core backend with generated controllers, typed service interfaces, and record models
- Kotlin Android ViewModel consuming the generated `VeldApi` singleton over HTTP
- In-memory data store — no database setup required

## Project layout

```
csharp-kotlin/
├── veld/                    ← Veld contracts (source of truth)
├── generated/               ← Auto-generated by `veld generate` (do not edit)
│   ├── Models/              — User.cs, Todo.cs, CreateUserInput.cs, etc.
│   ├── Services/            — IUsersService.cs, ITodosService.cs
│   └── Controllers/         — UsersController.cs, TodosController.cs
├── backend/
│   ├── VeldExample.csproj
│   ├── Program.cs
│   └── Services/
│       ├── UsersService.cs
│       └── TodosService.cs
└── frontend/
    └── TodoViewModel.kt
```

## Setup

### 1. Generate the code

```bash
cd csharp-kotlin/veld
veld generate
```

This writes typed record models, service interfaces, and ASP.NET controllers into `../generated/`.

### 2. Start the backend

```bash
cd csharp-kotlin/backend
dotnet run
```

The server listens on `http://localhost:5000`.

### 3. Use the Kotlin client

Copy `generated/client/ApiClient.kt` into your Android project's source tree (or add
it as a Gradle module). Set `VeldApi.baseUrl` to your running backend's address:

```kotlin
VeldApi.baseUrl = "http://10.0.2.2:5000"   // Android emulator → host loopback
```

Then use it from a ViewModel as shown in `frontend/TodoViewModel.kt`.

### Android dependencies

Add to your `build.gradle.kts`:

```kotlin
dependencies {
    implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.6.0")
}
```

Apply the serialization plugin:

```kotlin
plugins {
    kotlin("plugin.serialization")
}
```
