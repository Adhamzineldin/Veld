# python-vue

Flask + Vue 3 example powered by Veld.

## What this shows

- Python/Flask backend with generated route registration and typed ABC service interfaces
- Vue 3 frontend using `<script setup>` and the generated `api` client
- In-memory data store so no database setup is needed

## Project layout

```
python-vue/
├── veld/               ← Veld contracts (source of truth)
├── generated/          ← Auto-generated by `veld generate` (do not edit)
├── backend/
│   ├── app.py          ← Flask entry point
│   ├── requirements.txt
│   └── services/
│       ├── users_service.py
│       └── todos_service.py
└── frontend/
    └── UseTodos.vue
```

## Setup

### 1. Generate the code

```bash
cd python-vue/veld
veld generate
```

This writes typed models, ABC interfaces, Flask route handlers, Pydantic schemas,
and the fetch SDK into `../generated/`.

### 2. Start the backend

```bash
cd python-vue/backend
pip install -r requirements.txt
python app.py
```

The server listens on `http://localhost:5000`.

### 3. Start the frontend

Create a Vite + Vue 3 project in `frontend/`, install dependencies, then drop
`UseTodos.vue` in as a component.

```bash
npm create vite@latest frontend -- --template vue-ts
cd frontend
npm install
npm run dev
```

Set `VELD_API_URL=http://localhost:5000` (or configure `baseUrl` in
`veld.config.json`) so the generated SDK points at your running backend.
