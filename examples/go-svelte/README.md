# go-svelte

Go (Chi router) + Svelte 5 example powered by Veld.

## What this shows

- Go backend using Chi with generated route registration and typed service interfaces
- Svelte 5 frontend using `$state` runes and the generated store factory functions
- In-memory data store so no database setup is needed

## Project layout

```
go-svelte/
├── veld/               ← Veld contracts (source of truth)
├── generated/          ← Auto-generated by `veld generate` (do not edit)
├── backend/
│   ├── go.mod
│   ├── main.go
│   └── services/
│       ├── users.go
│       └── todos.go
└── frontend/
    └── TodoApp.svelte
```

## Setup

### 1. Generate the code

```bash
cd go-svelte/veld
veld generate
```

This writes Go structs, service interfaces, Chi route handlers, and the TypeScript fetch SDK into `../generated/`.

### 2. Start the backend

```bash
cd go-svelte/backend
go mod tidy
go run .
```

The server listens on `http://localhost:8080`.

### 3. Start the frontend

Create a SvelteKit or Vite + Svelte project in `frontend/`, then drop `TodoApp.svelte` in.

```bash
npm create vite@latest frontend -- --template svelte-ts
cd frontend
npm install
npm run dev
```

Set `VELD_API_URL=http://localhost:8080` (or configure `baseUrl` in `veld.config.json`) so the generated SDK points at your running backend.
